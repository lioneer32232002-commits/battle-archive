# 戰役模擬網站 + 中途島戰役 3D 頁面 — 設計文件

日期:2026-06-11
狀態:已獲使用者核准

## 目標

建立一個「歷史戰役模擬」靜態網站,以電視特別節目風格的 3D 運鏡與特效呈現戰役過程。
第一個戰役頁面為中途島戰役(1942/6/4 為主)。程式碼存於 GitHub,部署至 Cloudflare Pages。

## 網站結構

```
/                      主網站入口:選時期 → 戰役卡片列表(資料來自 battles.json)
/battles/midway/       中途島戰役 3D 模擬頁面
```

- 戰役清單為 JSON 資料驅動,未來新增戰役 = 新增一筆資料 + 一個資料夾。
- 每個戰役頁面是獨立的 Three.js 應用;共用工具日後逐步抽出,不預先做通用引擎(YAGNI)。

## 技術選型(方案 A,已核准)

- Vite + 原生 JavaScript + Three.js(npm 套件),多頁面建置(main + midway)。
- 全程序化生成模型與貼圖(canvas texture),不使用外部 3D 模型檔。
- 測試:時間軸引擎邏輯用 Vitest 單元測試;視覺以瀏覽器實際驗證。
- 介面語言:繁體中文為主,艦名/人名附原文。

## 視覺風格(已核准)

- 船艦做到「可辨識、有特徵」的中等細節程序化模型即停:
  - 赤城/飛龍艦島在左舷;加賀巨大船體;美軍航艦輪廓區別。
  - 日軍飛行甲板繪日之丸與片假名識別字(ア赤城、カ加賀、ソ蒼龍、ヒ飛龍)。
  - 艦上掛旗:日軍旭日軍艦旗(紅方)、美軍國旗(藍方)。
- 工夫投資在環境效果:動態海面 shader、雲層(6/4 多雲天氣)、爆炸火焰黑煙、
  魚雷航跡、防空曳光彈、俯衝轟炸彈道。
- 比例壓縮:海域依地理相對位置,船艦放大數百倍(電視節目標準做法)。

## 中途島頁面功能

1. **3D 場景**:海面、天空雲層、中途島環礁(沙島、東島、跑道)。
2. **時間軸控制**:播放/暫停/變速/拖曳。主軸 6/4 04:30–17:30(中途島當地時間),
   開頭簡述 6/3 前哨,結尾簡述 6/5–6/7(飛龍/赤城沉沒、三隈沉沒、約克鎮沉沒)。
3. **軍勢顯示**:
   - 紅方(日本):南雲機動部隊(赤城、加賀、蒼龍、飛龍 + 護衛)、指揮官南雲忠一、山口多聞。
   - 藍方(美國):TF16(企業號、大黃蜂號,史普魯恩斯)、TF17(約克鎮號,佛萊徹)、中途島基地航空隊;總指揮尼米茲。
   - 各部隊面板:指揮官、艦艇組成、艦載機戰力,戰損隨時間軸即時更新。
4. **重大事件**:字卡 + 導演運鏡(日軍空襲中途島、雷擊機隊全滅、10:22–10:28
   「命運的五分鐘」、飛龍反擊、約克鎮中彈、飛龍被擊毀…)。
5. **攝影機**:自由模式(OrbitControls)+ 導演模式(事件自動運鏡),可切換。

## 程式架構

- 時間軸引擎(純邏輯,可測試):時間→各單位位置插值、事件觸發、戰力結算。
- 戰役資料(JSON/JS 資料模組):單位、航跡 waypoints、事件、戰力數值。
- 3D 呈現層:場景、模型工廠、特效系統、攝影機導演。
- UI 層:時間軸控制列、部隊面板、事件字卡、圖例。

## 史料依據

時間軸與戰力數字以權威資料為準:美國海軍歷史與遺產司令部(NHHC)、
Parshall & Tully《Shattered Sword》、美軍戰鬥報告。實作時對關鍵時刻
(攻擊波時間、命中時間、各艦沉沒時間、艦載機數)做網路核對。
